Ensino de Programação e Linguagens de Programação

Dado que ninguém está cursando Computação ou algo parecido com isso eu vou falar brevemente sobre os desafios de ensinar programação. Muitas estratégias já foram sugeridas para melhorar o ensino de programação básica, dentre elas escolher a linguagem de programação "correta", o ponto é que não existe um consenso de qual seria a linguagem correta, mas o que se sabe é que quanto mais alto nível é a linguagem mais fácil é para aprender, por outro lado tem pesquisadores que defendem que não deveria nem ser usada linguagem de programação, já que programação é diferente de linguagem de programação, uma coisa é a prática a outra é a ferramenta. A taxa de reprovação nos cursos de programação básica no ensino superior são altas e é por isso que existe toda essa pesquisa em torno desse tema, eu não diria que é algo difícil de aprender, essa matéria geralmente é dada no 1° semestre da faculdade, os alunos ainda não se adptaram ao ritmo da faculdade e por isso não conseguem se organizar direito, enfim, existem vários fatores que contribuem para essa taxa de reprovação. Apesar de eu ter passado alguns anos lendo artigos científicos sobre esse tópico, eu não acho que tenho a melhor estratégia, mas posso garantir que tem embasamento científico e não foi tirado da minha cabeça. Por fim vale dizer que a melhor forma de aprender programação é a prática, sempre foi assim, não tem segredo.

O que é programação?

OK, falei um monte de coisa de linguagens de programação, disse até que é uma coisa separa da programação, mas não disse o que é programação.

Bom, eu já falei um pouco sobre isso na nota 00.html, mas vou repetir: No geral programação tem muito haver com entender as regras do mundo e conseguir descrever elas para um computador. Tem coisas fascinantes como ensinar um computador a falar ou a dirigir. Você acha que é fácil dirigir? Se você acha que essa é uma tarefa fácil é porque nunca pensou a fundo sobre isso, mas quando você precisa ensinar um computador a dirigir você percebe o quão complexo é essa tarefa. Da mesma forma conversar com outras pessoas, parece fácil, mas você consegue explicar para um computador como conversar com outras pessoas? Às vezes o computador me lembra uma criança de 5-6 anos de idade, se você não consegue explicar para uma criança de forma clara e correta como uma coisa funciona, então você não sabe de verdade como essa coisa funciona.

Dito isso, tem regras mais fáceis de programar e outras mais dificeis. É mais fácil programar regras que que são bem repetitivas, por exemplo uma calculadora, não importa quantas vezes você some 2 + 2 o resultado é sempre 4 a não ser que sua calculadora esteja quebrada. É diferente de uma conversa entre pessoas, você pode falar algo para uma pessoa e ter uma resposta, e pode falar essa mesma coisa para outra pessoa e ter uma resposta diferente, ou pior, você falar a mesma coisa para a mesma pessoa uma segunda vez e ter uma resposta diferente, parece compltamente imprevisível. A questão que fica é a mesma que eu carrego desde que comecei a programar: é imprevisível mesmo, e portanto impossível de programar, ou existe um padrão que não é fácil de enxergar? A Ciência sempre funcinou assim, hoje nós temos respostas para coisas que 500 anos pareciam imprevisíveis. Eu levanto esse ponto porque não é só uma conversa entre duas pessoas que parece imprevisível, existem outros comportamentos na natureza que parecem não seguir um padrão, mas talvez o padrão exista, só não seja elementar de enxergar.

Programas que lidam com imprevibilidade são os mais complexos de implementar, geralmente é usado algum modelo de Inteligência Artificial, existem alguns modelos, mas a ideia é que a partir de conjunto gigante de dados você consiga treinar uma IA para fazer o que os dados mostram. Por exemplo existe um caso interessante de uma IA que faria o mesmo papel de um Juíz, basicamente você treina esse IA com os dados de todos os julgamentos que já foram feitos dentro do país e o resultado de cada julgamentos, assim quando aparecer um novo caso a IA vai classificar o caso baseado nos dados históricos, e calcular a probabilidade desse novo caso o réu ser inocente ou culpado. Tem uma característica presente em todo programa que eu quero que você guarde: todo programa recebe dados de entrada, faz um processamento e retorna uma saída. Parece confuso eu sei, mas mais importante que isso é perceber que esse comportamento não é exatamente de um programa, é de tudo que acontece ao seu redor, o programa é só uma implementação de um comportamento.

  • Usando novamente uma conversa entre duas pessoas, quando você fala com outra pessoa, você deu um dado de entrada para ela por meio da fala, essa pessoa vai processar o que você falou e te dar uma resposta ainda que essa resposta seja o silêncio. Sim, até o silêncio é uma resposta.
  • Uma calculadora: coloca dois números e uma operação de entrada ela vai fazer um processamento e de dar o resultado da operação.
  • Para dirigir um carro você tem vários dados de entrada vindo a todo momento tanto na sua visão como audição. Você faz o processamento de acordo com a entrada. É gerada uma saída que pode ser acelerar ou freiar ou virar.
  • Uma pesquisa no Google recebe uma frase de entrada , faz um processamento buscando por sites com essa frase e te retorna uma saída que é uma lista de links
  • Já que falamos sobre juíz: um juíz recebe um dado de entrada que é um novo caso, faz um processamento , que é consultar a lei (que nesse caso está memorizada dentro da cabeça dele) e classificar o caso. Gera uma saída que é em favor ou desfavor de alguém.

Eu poderia dar muito mais exemplos, não sei se você já tinha reparado isso, mas toda ação ao seu redor pode ser resumida em: entrada -> processamento -> saída, só que nem sempre de uma maneira muito prevísivel como eu falei anteriormente. O importante é saber que a saída sempre depende da entrada. Se sua entrada for "curva a esquerda", sua saída deve ser "virar o volante para a esquerda", mas se sua entrada for "curva para a direita" sua saída deve ser "virar o volante para a direita".

Dado que a saída depende da entrada eu quero apresentar o primeiro conceito de uma linguagem de programação. Se chama variável, a variável é a estrutura da linguagem de programação para armazenar a entrada. Um programa bem simples seria um programa que diz "Olá" + o nome da pessoa. Então se a entrada for "Danilo" eu espero que a saída seja "Olá Danilo". Só que eu quero dizer "Olá" para qualquer pessoa, dependendo do valor da entrada, certo? Esse valor "nome" é variável, então nós temos aqui uma variável chamada nome. OK, mas para que serve esse programa? Para absolutamente nada, a não ser que você consiga vender ele. Nós estamos só começando, então aos poucos vai ficando mais complexo.
