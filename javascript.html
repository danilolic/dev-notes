0 - Comentários

Tem uma ferramenta super útil que eu ainda não falei. O comentário. Toda linguagem tem a possibilidade de fazer comentários no código, até o HTML e o CSS, mas não é muito utilizado nessas linguagens, só que aqui no Javascript vai ser mais utilizado. Para que serve? Para comentar! Não tem nenhuma diferença para o computador, o computador simplesmente ignora os comentários, logo, os comentários são usados para que outros programadores entendam melhor o seu código, você pode escrever um pedaço de código e escrever um comentário explicando para que serve esse pedaço de código, exatamente como eu fiz no exemplo abaixo.

1 - Bibliotecas

Toda linguagem de programação tem funções prontas para alguma coisa, geralmente coisas bem pequenas e simples, assim você não precisa ficar reinventando a roda toda vez que for programar, esses funções prontas são colocadas dentro de bibliotecas de acordo com o que ela faz, por exemplo você pode ter uma biblioteca para quando o usuário digitar coisas no teclado, você pode ter uma biblioteca para imprimir valores na tela, cada biblioteca é separada por funcionalidade, ou seja, todas as funções dentro de uma biblioteca estão juntas porque elas tem algo em comum que é o que caracteriza a biblioteca. No Javascript eu não tenho bons exemplos, mas vamos pegar a Linguagem C que com sorte vocês nunca vão usar, lá tem um biblioteca chamada Math (matemática), dentro dela tem funções para exponenciação, logarítmo, raíz quadrada, etc. Claro você poderia fazer suas próprias funções, mas não faz sentido já que você pode usar das bibliotecas. Daí logo de cara já tenho que falar sobre uma particularidade do Javascript, para algumas bibliotecas você precisa chamar usando o require e salvar ela numa variável. Isso acontece para algumas bibliotecas, outras nem precisa dar require , como por exemplo o console. Bom, não é tão importante assim porque para usar o Javascript com HTML não precisa de bibliotecas, vocês vão ver isso depois.

2 - Saída

Por enquanto no Javascript a saída vai ser sempre usando o console, mas saiba que o motivo do Javascript ser amplamente utilizado é porque ele permite manipular dados que estão numa página HTML, você pode adicionar, alterar ou remover dados de uma página HTML usando Javascript.

3 - Tipos de dados

Importante: dependendo do que você está construindo, sua entrada pode ter tipos diferentes, para um programa que busca palavras em um texto, sua entrada é um texto, para uma calculadora, sua entrada pode ser números. Essa pode não ser uma distinção clara para você, mas é uma distinção necessária para linguagens de programação, portanto toda linguagem de programação precisa que você programador(a) defina qual o tipo do dado. O Javascript tem dois tipo básicos: String e Number, sendo String o tipo usado para texto, ainda que esse texto tenha um único caractere; ou pode ser Number , ainda que seja inteiro ou decimal. Isso é importante porque um número pode ser representado em formato de String, já que você poderia muito bem ter um número dentro de um texto, daí se você tenta somar dois números que estão sendo representados como String não vai funcionar. No Javascript tudo que está sendo representado dentro de aspas, seja ela simples ou dupla ('') ou ("") é considerado String. O Javascript tem dois tipos mais complexos. A lista que é chamada também de Array, que serve para armazenar uma lista de itens (quem diria ein), poderia ser por exemplo uma lista de números de telefone ou qualquer outra lista que você imaginar, na prática para a lista de telefones você vai ter cada número de telefone sendo armazenado como String (texto), então no fundo, para o Javascript você está armazenando uma lista de Strings, quem dá significado para ela é o programador, não a linguagem. A forma de usar é assim:
let telefones = ["(61) 97430-0325", "(61) 3292-0284", "(61) 2130-2348"]
poderia ser uma lista de números que eu gosto (sem criatividade)
let numeros = [5, 7, 11, 13.02]

O outro tipo complexo é o objeto, ele serve para agrupar valores que de alguma forma fazem sentido ficar juntos. Por exemplo, pessoa:
let pessoa = { nome: "Danilo", idade: "desconhecida", signo: 'Aquário', numeros: [5, 7, 11, 13.02]}
Você poderia ter as variáveis: let nome, let idade, let signo, let numeros, mas usando objeto fica mais consistente, fica claro como esses dados estão relacionados. Perceba que você pode ter uma lista dentro de um objeto.

Da mesma forma que você pode ter uma lista dentro de um objeto, você pode também ter uma lista de objetos, nesse caso o objeto descreve uma pessoa, logo você teria uma lista de pessoas.
let pessoas = [
{ nome: "Danilo", idade: "desconhecida", signo: 'Aquário', numeros: [5, 7, 11, 13.02]},
{ nome: "Gabi", idade: "desconhecida", signo: 'Escorpião', numeros: [10, 13]}
]

Perceba que listas e objetos, ainda são formados pelos tipos básicos no final das contas (String e Number)
